  <action>
1. In `src/app/layout.jsx`, add `alternates: { canonical: './' }` to the existing metadata export. This sets a default canonical URL for all pages using the metadataBase. Next.js will resolve './' to the current page's URL path automatically.

2. In `src/app/page.jsx`, add a static metadata export above the default function:
```
export const metadata = {
  title: 'Ethos — What You Choose to Answer Reveals What You Stand For',
  description: 'A thought leadership platform where experts answer one curated question per day. Browse expert perspectives on today\'s most important questions.',
  alternates: { canonical: '/' },
}
```
Note: The homepage title uses the full `title` (not the template) because it's the brand page. Set `title` as a plain string to override the template.

3. In `src/app/login/page.jsx`, check if metadata exists. If not, add:
```
export const metadata = {
  title: 'Sign In',
  description: 'Sign in to Ethos with Google or LinkedIn.',
}
```

4. For all pages that already have static metadata (questions, experts, trending, leaderboard, topics, following, upcoming, privacy, terms), no changes needed — Next.js generates canonical from metadataBase + route path automatically once layout.jsx has the default `alternates.canonical`.

5. For dynamic pages with `generateMetadata` (q/[slug], answers/[id], expert/[handle], topics/[slug], search), add `alternates: { canonical: '/path/to/page' }` to each returned metadata object. Use the dynamic path:

   - topics/[slug]: `alternates: { canonical: \`/topics/\${slug}\` }`
   - search: `alternates: { canonical: '/search' }` (no query params in canonical)
