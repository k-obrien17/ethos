1. In `src/app/q/[slug]/page.jsx`, add a JSON-LD script tag inside the returned JSX of the `QuestionPage` component (at the top of the return, before other content). Use the data already fetched in the component.

Build a QAPage schema object:
```jsx
const BASE_URL = process.env.NEXT_PUBLIC_SITE_URL || 'https://ethos-daily.vercel.app'

const jsonLd = {
  '@context': 'https://schema.org',
  '@type': 'QAPage',
  mainEntity: {
    '@type': 'Question',
    name: question.body,
    text: question.body,
    dateCreated: question.publish_date,
    answerCount: answers?.length || 0,
    ...(answers?.length > 0 && {
      acceptedAnswer: {
        '@type': 'Answer',
        text: answers[0].body?.slice(0, 500),
        dateCreated: answers[0].created_at,
        author: {
          '@type': 'Person',
          name: answers[0].profiles?.display_name,
          url: `${BASE_URL}/expert/${answers[0].profiles?.handle}`,
        },
      },
      suggestedAnswer: answers.slice(1).map(a => ({
        '@type': 'Answer',
        text: a.body?.slice(0, 500),
        dateCreated: a.created_at,
        author: {
          '@type': 'Person',
          name: a.profiles?.display_name,
          url: `${BASE_URL}/expert/${a.profiles?.handle}`,
        },
      })),
    }),
  },
}
```

Add the script tag at the top of the returned JSX (inside a fragment if needed):
```jsx
<script
  type="application/ld+json"
  dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }}
/>
```

Important: The `answers` variable should already exist in the component from the data fetch. Locate where the component queries answers and use that variable. If the query only fetches answer IDs or partial data, the JSON-LD should use whatever fields are available (body, created_at, profiles.display_name, profiles.handle).
