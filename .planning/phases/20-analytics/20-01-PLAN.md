    **Sections to build (in order):**

    1. **Overview stats (keep existing):** Total users, total answers, total questions, total follows — already there, keep as-is.

    2. **Daily Active Users section (NEW):**
       - Query `answers` table grouped by `created_at::date` for the last 14 days to get unique expert_ids per day as "active users" (users who submitted answers).
       - Use Supabase RPC or raw SQL via `.rpc()` is not available for custom aggregation, so use multiple queries: for each of the last 7 days, count distinct expert_id from answers where created_at is within that day. To keep it efficient, fetch all answers from the last 14 days with `select('expert_id, created_at')` and group in JS.
       - Display today's DAU prominently, plus a simple table/list showing last 7 days of DAU.

    3. **Answer Submission Rates (NEW):**
       - Show answers per day (last 7 days), per week (last 4 weeks), per month (last 3 months).
       - Calculate by fetching answer created_at timestamps and grouping in JS.
       - Display as stat cards: "Today's Answers", "This Week", "This Month" plus comparison sublabels like "vs last week".

    4. **Expert Engagement (NEW):**
       - Fetch top 10 experts by answer count: query `answers` grouped by expert_id, count, join with profiles for display_name/handle.
       - For each, show: answer count, total likes received (sum of like_count on their answers), total views (sum of view_count).
       - Use Supabase: fetch answers with `select('expert_id, like_count, view_count')`, aggregate in JS, then fetch profile display_names for the top 10 expert_ids.

    5. **Most Popular Questions (NEW):**
       - Query questions that are published, join with answers to get answer_count and total_views (sum of answer view_counts per question).
       - Rank by total views + answer count.
       - Show top 10: question body (truncated to 80 chars), answer count, total views.
       - Approach: fetch all answers with `select('question_id, view_count')`, aggregate per question_id, then fetch question details for top 10.

    6. **Most Active Experts (NEW — enhanced from existing "Top Experts by Followers"):**
       - Replace current "Top Experts by Followers" with "Most Active Experts" showing: answer count, total likes, total views, follower count.
       - Sort by answer count (primary), likes (secondary).

    **Data fetching approach:**
    - Fetch bulk data in Promise.all at the top of the component (like existing pattern).
    - Fetch `answers` with `select('expert_id, question_id, view_count, like_count, created_at')` — this is the main dataset.
    - Aggregate in JS for DAU, submission rates, expert engagement, popular questions.
    - Keep recent signups section from existing page.

    **Styling:** Follow existing admin page patterns — section headers use `text-sm font-semibold text-warm-500 uppercase tracking-wide mb-3`, cards use `bg-white border border-warm-200 rounded-lg`, grid layouts for stat cards.
