The existing addComment server action checks auth but doesn't verify the user has a profile (i.e., completed the invite flow). Add a profile existence check as defense-in-depth.

In `src/app/actions/comments.js`, in the `addComment` function, after the auth check (`if (!user) return ...`), add a profile existence verification:

```
const { data: profile } = await supabase
  .from('profiles')
  .select('id')
  .eq('id', user.id)
  .single()

if (!profile) return { error: 'You must complete your profile to comment.' }
```

This ensures that even if someone bypasses the invite flow (e.g., direct API call with a valid Supabase auth token but no profile), they cannot post comments.
