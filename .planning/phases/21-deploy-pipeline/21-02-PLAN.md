<task type="auto">
  <name>Task 2: Configure GitHub branch protection rules</name>
  <files></files>
  <action>
Configure branch protection on the `main` branch using the GitHub CLI (`gh`). This enforces that PRs cannot be merged unless CI checks pass.

Run the following command:
```bash
gh api repos/{owner}/{repo}/branches/main/protection \
  --method PUT \
  --input - << 'EOF'
{
  "required_status_checks": {
    "strict": true,
    "contexts": ["lint-and-build"]
  },
  "enforce_admins": false,
  "required_pull_request_reviews": null,
  "restrictions": null
}
EOF
```

This sets:
- `required_status_checks.contexts: ["lint-and-build"]` — requires the CI job to pass
- `strict: true` — branch must be up to date before merging
- `enforce_admins: false` — admin can bypass in emergencies
- No required reviews (solo developer)
- No push restrictions

If `gh` is not authenticated, fall back to documenting the manual steps:
1. Go to GitHub repo > Settings > Branches > Add rule
2. Branch name pattern: `main`
3. Check "Require status checks to pass before merging"
4. Search for and select "lint-and-build"
5. Check "Require branches to be up to date before merging"
6. Save changes

Verify by running: `gh api repos/{owner}/{repo}/branches/main/protection` and confirming `required_status_checks` includes "lint-and-build".
  </action>
  <verify>
    <automated>gh api repos/:owner/:repo/branches/main/protection 2>/dev/null | python3 -c "import sys,json; d=json.load(sys.stdin); checks=d.get('required_status_checks',{}).get('contexts',[]); assert 'lint-and-build' in checks, f'Missing lint-and-build in {checks}'; print('PASS: Branch protection configured')" 2>/dev/null || echo "MANUAL CHECK: Visit GitHub repo > Settings > Branches to confirm main branch protection with lint-and-build required"</automated>
  </verify>
  <done>GitHub branch protection on `main` requires the "lint-and-build" CI check to pass before merge</done>
</task>
