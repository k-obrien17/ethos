  <files>next.config.mjs</files>
  <action>
  Add an `async headers()` function to `next.config.mjs` that returns Cache-Control headers for static assets:

  1. For `/fonts/:path*` and other public directory assets — set `Cache-Control: public, max-age=31536000, immutable` for any font files served from /public.

  2. For `/_next/static/:path*` — Vercel already sets immutable headers for hashed static assets, but add explicit headers as a safety net for non-Vercel deployments: `Cache-Control: public, max-age=31536000, immutable`.

  3. For `/favicon.ico` and `/icons/:path*` — set `Cache-Control: public, max-age=86400, stale-while-revalidate=604800` (1 day fresh, 1 week stale).

  The final config should look like:
  ```js
  const nextConfig = {
    images: { remotePatterns: [...] },
    async headers() {
      return [
        {
          source: '/_next/static/:path*',
          headers: [{ key: 'Cache-Control', value: 'public, max-age=31536000, immutable' }],
        },
        {
          source: '/favicon.ico',
          headers: [{ key: 'Cache-Control', value: 'public, max-age=86400, stale-while-revalidate=604800' }],
        },
      ]
    },
  }
  ```

  Note: next/font/google fonts are already served from /_next/static with immutable hashes. No /public font files exist in this project. The favicon and static assets are the primary targets.
