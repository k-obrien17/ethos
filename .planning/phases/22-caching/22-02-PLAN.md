  <action>
  Create `src/lib/supabase/cached.js` with cached query helpers using `unstable_cache` from `next/cache`.

  Important: `unstable_cache` caches the return value of an async function. The cached function cannot use cookies (so no auth-dependent queries). This is fine for public data like topics and site settings.

  Create a Supabase admin client import for cached queries (no cookie dependency):
  - Import `createClient` from `@/lib/supabase/admin` (already exists — uses service role key, no cookies)

  Define these cached helpers:

  1. `getCachedTopics()` — wraps a query for all topics with `select('id, name, slug')` ordered by name. Cache tag: `'topics'`. Revalidate: 300 (5 minutes).

  2. `getCachedSiteSettings(key)` — wraps a query for a single site_settings row by key. Cache tag: `'site-settings'`. Revalidate: 300 (5 minutes). Returns the `value` field or null.

  Pattern:
  ```js
  import { unstable_cache } from 'next/cache'
  import { createClient } from '@/lib/supabase/admin'

  export const getCachedTopics = unstable_cache(
    async () => {
      const supabase = createClient()
      const { data } = await supabase
        .from('topics')
        .select('id, name, slug')
        .order('name')
      return data ?? []
    },
    ['topics-list'],
    { revalidate: 300, tags: ['topics'] }
  )

  export const getCachedSiteSettings = unstable_cache(
    async (key) => {
      const supabase = createClient()
      const { data } = await supabase
        .from('site_settings')
        .select('value')
        .eq('key', key)
        .maybeSingle()
      return data?.value ?? null
    },
    ['site-settings'],
    { revalidate: 300, tags: ['site-settings'] }
  )
  ```

  Check `src/lib/supabase/admin.js` first to confirm the admin client exists and its export signature. If it uses `createClient` as async function, adjust accordingly. If admin.js doesn't exist or requires cookies, create a simple non-cookie Supabase client inline using `@supabase/supabase-js` directly (not `@supabase/ssr`):
  ```js
  import { createClient as createSupabaseClient } from '@supabase/supabase-js'
  const supabase = createSupabaseClient(process.env.NEXT_PUBLIC_SUPABASE_URL, process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY)
  ```
