- `created_at` TIMESTAMPTZ NOT NULL DEFAULT now()

**A. Server actions (`src/app/actions/topics.js`):**

Create with `'use server'` directive. All actions verify admin role (same pattern as questions.js):

1. `createTopic(prevState, formData)` — Takes `name` and optional `description`. Auto-generates `slug` from name (lowercase, replace spaces with hyphens, strip special chars). Validates name is 2-50 chars. Inserts into `topics` table. Revalidates `/admin/topics`. Returns `{ error }` or redirects.

2. `deleteTopic(topicId)` — Deletes topic by ID. CASCADE will remove question_topics entries. Revalidates `/admin/topics`, `/`, `/questions`.

3. `setQuestionTopics(questionId, topicIds)` — Accepts question UUID and array of topic UUIDs (0-3). Deletes all existing `question_topics` rows for this question, then inserts new rows. Validates array length <= 3. Revalidates relevant paths.

**B. Admin topics page (`src/app/admin/topics/page.jsx`):**

Server component. Fetches all topics ordered by name. Shows:
- Header "Topics" with count
- Create form: name input + optional description textarea + submit button. Use `useActionState` with `createTopic`. Make this a small client component embedded in the page (or extract to a separate `CreateTopicForm` client component inline).
- List of existing topics in a card: name, slug, description, question count (via joined query `topics(*, question_topics(count))`), delete button. Delete button calls `deleteTopic` server action.

Style consistently with existing admin pages (warm palette, rounded-lg borders, text-sm).

**C. Update admin layout (`src/app/admin/layout.jsx`):**

Add a "Topics" nav link pointing to `/admin/topics` alongside the existing Questions and Answers links.

**D. Update QuestionForm (`src/components/admin/QuestionForm.jsx`):**

Add a topic picker section to the form. The component is `'use client'`, so it needs topic data as a prop.

1. Add `topics` prop (array of `{ id, name }`) and `selectedTopicIds` prop (array of UUIDs for edit mode).
2. Add state: `const [selectedTopics, setSelectedTopics] = useState(selectedTopicIds ?? [])`.
3. Render a multi-select UI below the Category field:
   - Label: "Topics (up to 3)"
   - Show all available topics as clickable pills/chips. Selected ones are highlighted (bg-warm-800 text-warm-50), unselected are outlined (border-warm-300 text-warm-600).
   - Clicking a selected topic removes it. Clicking an unselected topic adds it (if < 3 selected).
   - Render hidden inputs: `<input type="hidden" name="topic_ids" value={selectedTopics.join(',')} />` so form submission includes them.
4. Show count: "X/3 selected"

**E. Update createQuestion and updateQuestion (`src/app/actions/questions.js`):**

After the question is inserted/updated, extract `topic_ids` from formData: `formData.get('topic_ids')`. If present and non-empty, split by comma, validate each is a UUID pattern, and call the logic to set question topics (delete existing + insert new, same as `setQuestionTopics` but inline to avoid circular imports). If empty string, clear all topics for the question.

**F. Pass topics to QuestionForm on create/edit pages:**

- Update `src/app/admin/questions/new/page.jsx`: fetch all topics, pass as prop to QuestionForm.
- Update `src/app/admin/questions/[id]/edit/page.jsx`: fetch all topics AND the question's current topic IDs (from question_topics), pass both to QuestionForm.

**G. Render topic pills on QuestionCard (`src/components/QuestionCard.jsx`):**

Add a `topics` prop (array of `{ name, slug }`). Render as small inline pills between category and the date/answer count footer. Style: `text-xs px-2 py-0.5 rounded-full bg-warm-100 text-warm-600 font-medium`. If no topics, render nothing extra.

**H. Update queries to include topics on all pages that render QuestionCard:**

1. **Homepage (`src/app/page.jsx`):** When fetching `todayQuestion` and `recentQuestions`, add a join to get topics: `.select('*, answers(count), question_topics(topics(name, slug))')`. Pass the extracted topics array to QuestionCard and to the today's question display.

2. **Question page (`src/app/q/[slug]/page.jsx`):** When fetching the question, join topics: `.select('*, question_topics(topics(name, slug))')`. Render topic pills in the question header area, next to category and date.

3. **Answer page (`src/app/answers/[id]/page.jsx`):** The answer query already joins `questions!inner(...)`. Extend the questions select to include topic data: `questions!inner(id, body, slug, category, publish_date, question_topics(topics(name, slug)))`. Render topic pills in the question context box.

4. **Questions list page (`src/app/questions/page.jsx`):** Add topic join to the questions query. Pass topics to QuestionCard.

5. **Admin questions page (`src/app/admin/questions/page.jsx`):** Add topic join to the questions query. Show topic names as small pills in the QuestionRow component.

For Supabase nested joins, the pattern is: `.select('*, question_topics(topics(name, slug))')`. The result will be `question.question_topics = [{ topics: { name, slug } }]`. Transform to `question.topics = question.question_topics?.map(qt => qt.topics) ?? []` before passing to components.

**Important notes:**
- Do NOT remove the existing `category` field — it remains separate from topics. Topics are a many-to-many taxonomy; category is a single freeform field.
- All topic pills should be non-interactive on public pages (just display, no links yet — links come in plan 10-02 with the topic browse page).
- Wrap topic pill rendering in a reusable pattern but don't over-engineer — inline JSX is fine for now.
