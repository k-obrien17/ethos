Extend `src/app/actions/search.js` with a new exported action for typeahead suggestions. This needs to be fast and return fewer results than the full search.

Add this function to the existing file:

```javascript
export async function searchSuggestions(query) {
  if (!query || query.trim().length < 2) {
    return { suggestions: [] }
  }

  const supabase = await createClient()

  // Use the same search_content RPC but with a smaller limit
  // and no filters for maximum suggestion coverage
  const { data, error } = await supabase.rpc('search_content', {
    search_query: query.trim(),
    filter_type: null,
    filter_topic_id: null,
    filter_date_range: null,
    result_limit: 7,
    result_offset: 0,
  })

  if (error) {
    return { suggestions: [] }
  }

  // Transform results into suggestion format
  const suggestions = (data ?? []).map(r => ({
    id: r.result_id,
    type: r.result_type,
    title: r.title,
    url: r.url,
    subtitle: r.result_type === 'answer' ? r.author_name :
              r.result_type === 'expert' ? `@${r.author_handle}` : null,
  }))

  return { suggestions }
}
```

The suggestions response is intentionally lightweight — just enough to display in the dropdown. No snippets needed (the title is sufficient for typeahead).

**A. SearchTypeahead component (`src/components/SearchTypeahead.jsx`):**

Client component (`'use client'`). Renders the dropdown panel below the search input with recent searches or live suggestions.

Props: `{ query, onSelect, onClose, inputRef }`

State:
- `suggestions` — array of typeahead results from server action
- `recentSearches` — loaded from localStorage
- `selectedIndex` — for keyboard navigation (-1 = nothing selected)
- `isLoading` — loading state for suggestions

Logic:
1. **Recent searches (when query is empty and focused):**
   - Read from `localStorage.getItem('ethos_recent_searches')` — parse as JSON array of strings, max 5 items
   - Display as a list with a clock icon and the search term
   - Clicking a recent search calls `onSelect` with that term (navigate to /search?q=...)
   - Show "Clear recent searches" link at bottom that clears localStorage

2. **Live suggestions (when query has 2+ chars):**
   - Debounce the query with 250ms delay before calling `searchSuggestions`
   - Use `useEffect` with a timer to debounce. On each keystroke, clear the previous timer and set a new 250ms timeout before calling the server action.
   - While loading, show a subtle spinner or skeleton
   - Display each suggestion as a row with:
     - Type icon (magnifying glass for questions, quote/chat bubble for answers, person icon for experts)
     - Title text (truncated to one line)
     - Subtitle text (author name for answers, handle for experts)
   - "See all results for '[query]'" link at the bottom always present

3. **Keyboard navigation:**
   - Arrow Down: increment `selectedIndex` (wrap to 0 at end)
   - Arrow Up: decrement `selectedIndex` (wrap to end at -1)
   - Enter: if an item is selected, navigate to its URL. If nothing selected, submit the form (navigate to /search?q=...)
   - Escape: call `onClose()`
   - The keyboard handler should be attached to the input element (passed via `inputRef`) — actually, handle it in `SearchBar` and pass relevant events down, or handle in the typeahead itself by listening for events

   Since the keyboard events need to happen on the input, handle them in `SearchBar` and pass the `selectedIndex` and items list to coordinate.

   Simplest approach: Let `SearchBar` manage keyboard navigation state and pass it to `SearchTypeahead`. Or, let `SearchTypeahead` expose the items and selected index, and `SearchBar` handles the keyboard.

   Cleanest approach: `SearchTypeahead` renders the dropdown and accepts `selectedIndex` prop (controlled from parent). `SearchBar` handles all keyboard events on the input and manages `selectedIndex` state. When Enter is pressed and an item is selected, `SearchBar` navigates to that item's URL.

4. **Rendering:**

**B. SearchBar component (`src/components/SearchBar.jsx`):**

Client component (`'use client'`). The persistent search input in the nav.

Props: none (self-contained)

State:
- `query` — current input value
- `isFocused` — whether the input is focused (controls typeahead visibility)
- `isExpanded` — for mobile: whether the search input is visible (default false on narrow screens)
- `selectedIndex` — keyboard nav index for typeahead items

Logic:

1. **Keyboard shortcut (Cmd+K / Ctrl+K):**
   ```javascript
   useEffect(() => {
     function handleGlobalKeyDown(e) {
       if ((e.metaKey || e.ctrlKey) && e.key === 'k') {
         e.preventDefault()
         inputRef.current?.focus()
         setIsFocused(true)
       }
     }
     document.addEventListener('keydown', handleGlobalKeyDown)
     return () => document.removeEventListener('keydown', handleGlobalKeyDown)
   }, [])
   ```

2. **Form submission:**
   On form submit (Enter key with no selected suggestion), navigate to `/search?q=...` and save the query to recent searches in localStorage:
   ```javascript
   function handleSubmit(e) {
     e.preventDefault()
     if (selectedIndex >= 0 && suggestions[selectedIndex]) {
       // Navigate to selected suggestion
       router.push(suggestions[selectedIndex].url)
     } else if (query.trim().length >= 2) {
       // Save to recent searches
       saveRecentSearch(query.trim())
       router.push(`/search?q=${encodeURIComponent(query.trim())}`)
     }
     setIsFocused(false)
     inputRef.current?.blur()
   }
   ```

3. **Recent searches localStorage helpers:**
   ```javascript
   function getRecentSearches() {
     try {
       return JSON.parse(localStorage.getItem('ethos_recent_searches') || '[]').slice(0, 5)
     } catch { return [] }
   }

   function saveRecentSearch(term) {
     const recent = getRecentSearches().filter(s => s !== term)
     recent.unshift(term)
     localStorage.setItem('ethos_recent_searches', JSON.stringify(recent.slice(0, 5)))
   }

   function clearRecentSearches() {
     localStorage.removeItem('ethos_recent_searches')
   }
   ```

4. **Click outside to close:**
   Use a ref on the container and a `mousedown` event listener to detect clicks outside. When clicked outside, set `isFocused = false`.

5. **Keyboard navigation on input:**
   ```javascript
   function handleKeyDown(e) {
     if (e.key === 'ArrowDown') {
       e.preventDefault()
       setSelectedIndex(prev => Math.min(prev + 1, totalItems))
     } else if (e.key === 'ArrowUp') {
       e.preventDefault()
       setSelectedIndex(prev => Math.max(prev - 1, -1))
     } else if (e.key === 'Escape') {
       setIsFocused(false)
       inputRef.current?.blur()
     }
   }
   ```
