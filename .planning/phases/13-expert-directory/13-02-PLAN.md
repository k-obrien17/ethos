1. **Migration** (`00023_featured_expert.sql`):
   Create a `site_settings` table for key-value config:
   ```sql
   CREATE TABLE public.site_settings (
     key TEXT PRIMARY KEY,
     value TEXT,
     updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
   );
   ALTER TABLE public.site_settings ENABLE ROW LEVEL SECURITY;
   CREATE POLICY "Site settings are publicly readable" ON public.site_settings FOR SELECT USING (true);
   CREATE POLICY "Admins can manage site settings" ON public.site_settings FOR ALL USING (
     EXISTS (SELECT 1 FROM public.profiles WHERE profiles.id = auth.uid() AND profiles.role = 'admin')
   );
   -- Seed the featured_expert key
   INSERT INTO public.site_settings (key, value) VALUES ('featured_expert_id', NULL);
   ```

2. **Admin experts page** (`src/app/admin/experts/page.jsx`):

   - Each expert row has a "Feature" button (SetFeaturedButton client component)
   - A "Clear Featured" button if someone is currently featured
   - Metadata: title "Admin: Experts"
