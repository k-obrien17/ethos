  <action>
Create `src/app/experts/page.jsx` — a server component at /experts that:

1. **Data fetching** (all parallel via Promise.all):
   - Fetch all profiles (id, handle, display_name, avatar_url, headline, organization, follower_count)
   - Fetch all answers (expert_id, question_id, created_at, like_count, view_count) for aggregation
   - Fetch all question_topics with topics (question_id, topic_id, topics(id, name, slug)) for expertise derivation
   - Fetch all topics for the filter dropdown
   - Fetch total questions this month for selectivity calculation (same pattern as expert/[handle]/page.jsx)

2. **Expert stats aggregation** (in JS, following leaderboard pattern):
   - For each expert_id in answers: compute answer_count, total_likes, most_recent_answer_date
   - For selectivity: count answers this month vs total questions this month, compute percentage
   - For topic expertise: map each expert's answers -> question_ids -> question_topics -> topic_ids, count per topic, take top 3 topics by answer count as expertise tags

3. **URL-driven sort/filter** (read from searchParams, following search page pattern):
   - Sort options: "answers" (answer count desc, default), "selectivity" (% desc), "recent" (most recent answer desc), "likes" (total likes desc)
   - Topic filter: optional topic slug from query param, filters to experts who have answered questions in that topic
   - Render sort controls as `<Link>` elements with active state styling (not client component — keep it server)

4. **Expert cards** — for each expert, render a card (Link to /expert/[handle]) with:
   - Avatar (img or initial fallback, same pattern as leaderboard)
   - display_name, headline or organization or @handle
   - Stats row: answer count, selectivity %, likes
   - Topic expertise pills: up to 3 topic pills styled like existing topic pills (text-xs px-2.5 py-1 rounded-md bg-warm-100 text-warm-600) linking to /topics/[slug]
   - Card styled like leaderboard entries: bg-white rounded-lg border border-warm-200 hover:border-warm-300

5. **Nav link**: Add "Experts" link to Header.jsx desktop nav (after "Topics" link, same styling). Add to MobileNav.jsx as well.

6. **Metadata**: export metadata with title "Expert Directory" and description.

Use `export const revalidate = 300` for ISR caching.

Empty state: "No experts have answered yet." centered message.
