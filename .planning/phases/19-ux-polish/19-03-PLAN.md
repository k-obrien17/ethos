    Add ARIA attributes and keyboard support to components. Read each file first, then make targeted additions:

    1. **src/app/layout.jsx** — Add a skip-to-content link as the FIRST child inside the body:
       ```jsx
       <a href="#main-content" className="skip-link">Skip to main content</a>
       ```
       And add `id="main-content"` to the main content container (the element wrapping {children}).

    2. **src/components/ReportButton.jsx** — Add `aria-label="Report this content"` to the button element. If it's an icon-only button, this is essential for screen readers.

    3. **src/components/DeleteAccountSection.jsx** — Add `aria-label="Delete your account"` to the delete button. Add `role="alertdialog"` and `aria-labelledby` to any confirmation dialog if present.

    4. **src/components/BudgetIndicator.jsx** — Add `aria-label` describing the budget state, e.g., `aria-label={`${remaining} of ${total} answers remaining today`}`. Add `role="status"` so screen readers announce budget changes.

    5. **src/components/QuestionCard.jsx** — Add `role="article"` to the card container div. Ensure the main link has descriptive text (the question title should suffice).

    6. **src/components/AnswerCard.jsx** — Add `role="article"` to the card container div. Ensure the expert name link has context.

    7. **src/components/NotificationBell.jsx** — Verify existing aria-label is descriptive. If it just says "notifications", enhance to include unread count: `aria-label={unreadCount ? `${unreadCount} unread notifications` : 'Notifications'}`.

    8. **src/components/MobileNav.jsx** — Verify the hamburger/menu toggle has `aria-label="Toggle navigation menu"` and `aria-expanded={isOpen}`. Ensure the nav overlay has `role="dialog"` and `aria-label="Navigation menu"`.

    Key principles:
    - Icon-only buttons MUST have aria-label
    - Dynamic state (counts, open/closed) should be reflected in aria attributes
    - Cards should use role="article" for semantic grouping
    - Do NOT add redundant labels to elements that already have visible text
