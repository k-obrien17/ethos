<task type="auto">
  <name>Task 2: Add toast confirmations to all interactive components</name>
  <files>
    src/components/AnswerForm.jsx

    src/components/FollowButton.jsx

    src/components/EditProfileForm.jsx
    src/components/ShareButton.jsx
  </files>
  <action>
    Add `import { toast } from 'sonner'` to each component and add toast calls at the appropriate points:

    1. **AnswerForm.jsx** — On successful answer submission: `toast.success('Answer saved')`. On error: `toast.error('Failed to save answer')`.

    2. **CommentSection.jsx** — On successful comment post: `toast.success('Comment posted')`. On successful comment delete: `toast.success('Comment deleted')`. On error: `toast.error('Failed to post comment')` or `toast.error('Failed to delete comment')`.

    3. **FollowButton.jsx** — On follow: `toast.success('Following {displayName}')`. On unfollow: `toast.success('Unfollowed {displayName}')`. On error: `toast.error('Failed to update follow')`.

    4. **FollowButtonSmall.jsx** — Same pattern as FollowButton but uses the same display name from props.

    5. **FollowTopicButton.jsx** — On follow: `toast.success('Following {topicName}')`. On unfollow: `toast.success('Unfollowed {topicName}')`. On error: `toast.error('Failed to update follow')`.

    6. **BookmarkButton.jsx** — On bookmark: `toast.success('Bookmarked')`. On remove: `toast.success('Bookmark removed')`. On error: `toast.error('Failed to update bookmark')`.

    7. **LikeButton.jsx** — NO toast for likes (too frequent/ephemeral — optimistic UI state change is sufficient feedback). Only add `toast.error('Failed to update like')` on error.

    8. **EditProfileForm.jsx** — On successful save: `toast.success('Profile updated')`. On error: `toast.error('Failed to update profile')`.

    9. **ShareButton.jsx** — On successful clipboard copy: `toast.success('Link copied to clipboard')`. On error: `toast.error('Failed to copy link')`.

    Integration pattern for each component:
    - Find the server action call (usually inside a startTransition or try/catch)
    - Add toast.success() after the successful action completes
    - Add toast.error() in the catch block or error handling path
    - Keep existing optimistic UI logic — toasts are ADDITIONAL feedback, not replacement
    - Use concise messages (2-4 words for success, slightly longer for errors)
  </action>
